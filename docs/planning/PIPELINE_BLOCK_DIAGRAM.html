Pipeline Architecture Plan

RF Drone Detection Pipeline Block Diagram

제한된 예산과 연산 자원에서 RF 후보를 단계적으로 좁히고, 드론으로 확정된 RF track에 대해서만 방향을 출력하는 soft real-time RF 추적 파이프라인입니다.

1. 설계 목적

이 문서는 단순한 선형 처리 흐름이 아니라, scan mode, candidate list, gated decision, conservative AoA output, track mode를 포함한 전체 아키텍처를 설명합니다.

이 시스템은 하드 리얼타임 제어 시스템이 아니라, 제한된 연산 자원에서 의미 있는 RF 후보에만 계산을 집중하는 soft real-time RF 추적 파이프라인입니다.

2. 핵심 설계 철학

Soft Real-time Tracking

단일 block 즉시 판정보다 짧은 block window 기반 안정 판정을 우선합니다.

  • 1 block 약 3.3ms
  • 7 blocks 약 23ms
  • 15 blocks 약 50ms
Electronic-first RF Sensing

모터, 기계식 sweep, PID motion control layer를 제외하고 전자식 scan과 RF 채널 확장을 우선합니다.

Compute-aware Gating

Energy Gate, CNN 안정성, Coherence Gate를 순차적으로 통과한 후보에만 무거운 계산을 수행합니다.

4. Scan Mode와 Candidate List

Scan Mode에서는 sweep 도중 Energy Detection이 발생해도 즉시 sweep을 중단하지 않습니다. 한 sweep은 끝까지 완료하고, energy가 검출된 center frequency/block은 candidate list에 등록합니다.

CNN 입력 spectrogram에는 신호 세기, 시간-주파수 패턴, burst 길이와 반복성이 포함됩니다. 따라서 energy pattern, spectrogram pattern, burst consistency는 별도 점수가 아니라 CNN evidence 내부의 하위 근거로 봅니다.

6. Decision Policy

상황동작방향 표시

판정 불안정watch 또는 candidate 재검사없음

Drone confirmed + coherence passPhase difference, AoA, azimuth 계산있음

AoA는 드론 판정 evidence가 아니라, 드론으로 확정된 RF track에만 제공되는 direction output입니다. WiFi/Bluetooth 같은 비드론 RF에는 각도를 표시하지 않습니다.

8. Compute Saving 구조

Naive 구조는 모든 block에 STFT, CNN, Coherence, AoA를 수행하지만, gated 구조는 후보가 줄어든 뒤에만 무거운 연산을 수행합니다.

C_gated =
N * C_energy
+ N * p_energy * (C_stft + C_cnn)
+ N * p_energy * p_drone * C_coherence
+ N * p_energy * p_drone * p_coherence * C_aoa
N total blocks전체 sweep 대상
N_activeEnergy Gate 통과
N_drone_candidateCNN/window 통과
N_coherentCoherence 통과
N_confirmedAoA 출력 대상

핵심 관계: N_confirmed << N_drone_candidate << N_active << N_total

11. 핵심 발표 문장

Scan Mode에서는 sweep 도중 energy가 검출되어도 즉시 중단하지 않고 candidate list에 등록한 뒤, sweep 종료 후 후보군을 상세 분석합니다. AoA는 드론 여부를 결정하기 위한 evidence가 아니라, 드론으로 확정되고 coherence gate를 통과한 RF track에 대해서만 제공되는 방향 출력입니다.
