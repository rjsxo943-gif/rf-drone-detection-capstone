Project Positioning

완성형 대드론 장비가 아니라,
탐지의 첫 계층을 직접 만드는 프로젝트

이 브랜치의 초점은 전체 시스템 완성보다 CNN 학습용 RF spectrogram 데이터 수집 파이프라인 안정화에 있습니다. 실제 SDR 입력, 저장된 IQ 파일, 시뮬레이션 입력을 같은 구조 안에서 다루며 재현 가능한 실험 흐름을 만드는 것이 핵심입니다.

Problem

눈에 보이지 않는 RF 신호

드론 의심 신호는 영상보다 먼저 RF 대역에서 포착될 수 있습니다. 2.4GHz 혼잡 환경에서는 Wi-Fi, Bluetooth, 배경 신호를 함께 구분해야 합니다.

Approach

IQ에서 Spectrogram으로

수신한 IQ block을 전처리하고 Energy Detector로 유효 block을 선별한 뒤, STFT spectrogram으로 변환해 CNN 입력으로 사용합니다.

Outcome

분류와 방향 추정의 연결

RF3 baseline은 비드론 RF 신호 분류 기반을 만들고, 2채널 입력이 확보되면 coherence 검사와 위상차 기반 AoA 추정으로 확장됩니다.

Signal Pipeline

RF IQ 수신부터 결과 저장까지 이어지는 처리 흐름

README 기준 파이프라인은 수신, 전처리, 탐지, 변환, 학습/추론, 2채널 방향 추정 branch로 분리됩니다. 현재 단일 채널 수집에서는 AoA 값이 비어 있을 수 있고, RX0/RX1 입력 확보 시 의미 있게 동작합니다.

RF3 CNN Baseline

Background, Bluetooth, WiFi를 구분하는 기준 모델

실제 드론 RF 데이터가 아직 확보되지 않았기 때문에, 현재 모델은 드론 탐지 모델이 아니라 2.4GHz 대역의 비드론 RF 신호를 구분하는 baseline CNN입니다. 대표 모델은 learning rate 0.001 조건에서 test set 225개 중 222개를 정확히 분류했습니다.

결과는 현재 수집 데이터셋 내부의 baseline 성능입니다. 새로운 장소, 거리, 주파수, 장비 조건에 대한 일반화 성능은 추가 검증이 필요합니다.

Roadmap

다음 단계는 데이터, 모델, 2채널 검증의 연결

CNN 학습용 spectrogram 데이터 수집 안정화

Pluto+ SDR 또는 저장된 IQ 파일에서 의미 있는 block을 선별하고 label별 데이터셋을 재현 가능하게 구성합니다.

RF3 baseline checkpoint를 runtime pipeline에 연결

학습된 CNN 모델을 실시간 또는 파일 기반 파이프라인에서 바로 사용할 수 있는 inference 구조로 정리합니다.

실제 drone-like 데이터 확보와 클래스 확장

현재 비드론 3분류 baseline에 드론 의심 신호 데이터를 추가해 탐지 문제로 확장합니다.

RX0/RX1 2채널 AoA branch 검증

coherence gate, phase difference, arcsin 기반 angle 변환을 실제 2채널 입력으로 검증합니다.

RF 신호를 데이터셋으로 만들고, 데이터셋을 탐지 파이프라인으로 연결합니다.

이 페이지는 `experiment/stft-128-hop32` 브랜치의 README와 RF3 CNN baseline 보고서를 기준으로 구성했습니다.
